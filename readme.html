Blog Pro

Version 1.2.3

Nowadays, it is impossible to imagine a website without blog, a powerful tool which helps inform your customers about whatever you want – news, discounts, sales, new products, events, etc. Especially if it’s SEO-friendly, blog influences search engines rankings.

MagPleasure is glad to announce the release of Blog Pro, a new extra-powerful Magento extension of excellent quality. Our module is developed so that you can easily present the information to your Magento website visitors as well as change the opinions with them.

Our Blog Pro Magento extension is the best SEO-friendly solution you have ever met. With our Magento module, you can be sure that your blog will work for you and bring new potential customers to your store from different search engines.

Installation

Before you start, we would like to point out that if you are not confident performing the installation yourself, you should let the team of Magpleasure Ltd. do it. People who developed the extension would be the best choice and they usually offer this service for a small fee. See http://www.magpleasure.com/installation-service.html.

Backup your data

Make a Backup copy of the files and database for your Magento store.

Disable Compilation

If you have compilation enabled, please disable it. Installing extension with enabled compilation will inevitably lead to problems in the front end or the backend. This can be corrected, but would require an expert help. So to avoid it, temporary disable the compilation.

Enable all Types of Cache

If you disable the cache, then turn it on and refresh it. It is best to install extensions to the store, when no one except you have access to it during that time, because installation process can begin before all extensions files are loaded. This may lead to some problems.

Download and unzip the package

Download and unzip the extension package from the website where you bought it from. On our site you can get this package from My Downloadable Products in the customer area.

Once the package is unpacked, go into the unpacked directory

Upload files to the server. Upload app, js and skin folders to the server using an FTP client. If you are using a modified theme or a custom template, you will need to make an additional copy of the directory app/design/frontend/default/default to app/design/frontend/<YOUR_PACKAGE>/<YOUR_THEME>.

After that copy skin/frontend/default/default to skin/frontend/<YOUR_PACKAGE>/<YOUR_THEME>.

Depending on the extension function, some folders may not exist. In this case, they do not need to be copied.

Additional installation steps

Some extensions require you to make additional modifications to the source files of your store. Find the Installation section in the readme.html file, which comes with each extension. If any, here you would find any additional modifications.

Refresh Store Cache

Unless you refresh the store cache, the extension will not be installed properly. Go to System → Cache Management, and click Flush Cache Storage. The cache is cleared and the extension will be installed to the store after refreshing of any page.

That's done

The extension is installed to your store and you can start configuring it if you need. If you were using compilation, you can enable it now.

Installation of MP-Culumus

We found MT-Cumulus a great tool and that’s why integrated it with our Blog Pro. Unfortunately, we can’t add this tool to our extension package, but you can download and install it yourself.

1. Just go to http://wordpress.org/extend/plugins/wp-cumulus/installation/

2. Find the Download the plugin link and download the MT-Cumulus package. Or just download the package using a direct link http://www.magpleasure.com/media/wp-cumulus/wp-cumulus.zip

3. Unpack the swfobject.js and tagcloud.swf files into the \js\mpblog\wp-cumulus\ folder.

Configuration

To set up the Blog Pro Magento extension go to the System → Configuration → Blog Pro page at the backend. Here you can configure the module in the way you need.

The Menu Integration section is responsible for the integration of the Blog Pro extension in the menu:

  • Enabled – switch on/off the link which leads to your blog from top and footer menu
  • Label – specify the link title
  • Position – specify the link position in menu

The Search Engine Optimization section is responsible for your blog optimization for search engines:

  • Route – the route to blog in the link. For example, http://store.com[route_to_blog].html
  • Title Prefix for Blog Pro – if Meta title is not specified for post or category, it is displayed as “[this_prefix] – [title_of_entity]”

The General section is responsible mainly for blocks displaying on different pages:

  • Blog Pro Layout – specify the layout of the blog page
  • Categories Block Position – define the position for the categories block displayed on the Blog page. If you don’t want to use categories at all, select "No" and they won’t be displayed neither on the Blog page, nor on the posts pages.
  • Tags Block Position – set up the tags block position on the Blog page. If you don’t want to use tags, select "No" and tags won’t be displayed neither on the Blog page, nor on the posts pages.
  • Recent Posts Block Position – define the position of the recent posts block. If you don’t want to use this block, just select "No".
  • Recent Comments Block Position – specify the position of the block with the latest comments on the Blog page. If you don’t want to use this block, just select "No".

The Comments Configuration section is responsible for the comments functioning:

  • Use Comments – enable/disable everything relates to comments
  • Automatical Approving – make the comments approved automatically
  • Allow Guests – allow not logged-in users to leave comments in your blog
  • Recent Comments Number in Block – specify the number of the latest comments displayed in the corresponding block

The Recent Posts Configuration section is responsible for the block with recent posts:

  • Recent Posts Number in Block - specify the number of the latest posts displayed in the corresponding block
  • Display Short Content – specify whether to display short description of the post or not
  • Short Content Limit – specify the length of the short description displayed in block

The Tags Configuration section is responsible for tags cloud:

  • Minimal Posts Number per Tag – define the minimal number of posts which use the same tag to make it displayed in the tags cloud on the Blog page
  • Use MT-Cumulus Tag Cloud – enable this option to make tags cloud displayed. Remember that MT-Cumulus must be installed at first
  • MT-Cumulus: Width – the width of the tags cloud
  • MT-Cumulus: Height – the height of the tags cloud (the same height as width is recommended)
  • MT-Cumulus: Text Color – the text color displayed in the tags cloud
  • MT-Cumulus: Background Color – the color of the tags cloud background

The RSS Configuration section is responsible for the RSS feed:

  • Post Feed – enable RSS for posts
  • Comment Feed – enable RSS for posts comments

The Social Configuration section is responsible for the Social Sharing block settings:

  • Enabled – switch on/off the Share and Enjoy block for blog posts
  • Networks – select which social buttons will be displayed in the Share and Enjoy block

After blog configuration, you can create posts and categories:

  • CMS → Blog Pro → Posts
  • CMS → Blog Pro → Categories

While a post adding, tags are created automatically. You can edit them at CMS → Blog Pro → Tags.

To approve and edit comments, go to CMS → Blog Pro → Comments.

To import data from other blog solutions, go to System → Blog Pro Tools → Import and select solution you need.

Reminder!

If you have already configured cron jobs for your Magento installation, then you can skip the following step. Read the following instructions on setting cron job for your Magento store: http://www.magentocommerce.com/wiki/1_-_installation_and_configuration/how_to_setup_a_cron_job Generally it would be enough to run in SSH console of your server: 

crontab -e

And insert the following line: 

*/3 * * * * php /<path-to-your-installation/cron.php

After that exit confirming save request.

Troubleshooting

After the extension installation the store gives an error, or blank page, or suggests to start Magento installation procedure.
Change the owner of the extracted extension files to the web server user and set 775 permissions on them. Clear the store cache and try again.
There is no MagPleasure extensions under my configuration section, or having the extension tab clicked I get a blank page, or Access Denied error.
Clear the store cache, browser cookies, logout and login again.
I've set up everything correctly, inserted the HTML code but there is nothing on that page.
Clear the store cache, clear your browser cache and domain cookies and refresh the page.
My configuration changes do not appear on the store.
Clear the store cache, clear your browser cache and domain cookies and refresh the page.
